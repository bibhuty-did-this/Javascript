Asyncronous Javascript

  • It means something is going on and you don't want to wait until that someting is completed and continue with your program
  • It is relevant to javascript as it may take some time to get data from server but we need to do other things in the mean time.
  • Callbacks were used for long time until ES6 arrived with promises.
  • Async Await was introduced in ES7 which still deals with promises but in an elegant way and looks more like synchronous programming.
